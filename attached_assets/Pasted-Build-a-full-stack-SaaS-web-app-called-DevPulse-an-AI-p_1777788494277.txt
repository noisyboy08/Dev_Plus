=== DATABASE SCHEMA ===
Table: users
  - id INTEGER PRIMARY KEY
  - github_id TEXT UNIQUE
  - username TEXT
  - avatar_url TEXT
  - access_token TEXT (encrypted)
  - created_at DATETIME

Table: standups
  - id INTEGER PRIMARY KEY
  - user_id INTEGER (FK в†’ users)
  - repo_name TEXT
  - date DATE
  - yesterday TEXT (AI generated)
  - today TEXT (AI generated)
  - blockers TEXT (AI generated JSON array)
  - next_task TEXT (AI generated)
  - raw_activity TEXT (JSON of commits/PRs)
  - sent_to_slack BOOLEAN DEFAULT false
  - created_at DATETIME

Table: preferences
  - user_id INTEGER PRIMARY KEY (FK в†’ users)
  - slack_webhook_url TEXT
  - standup_tone TEXT DEFAULT 'professional' (professional/casual/bullet-points)
  - default_repo TEXT

=== CORE FEATURES TO BUILD ===

1. GITHUB OAUTH LOGIN
   - Login with GitHub button on landing page
   - Store user + access_token in DB and session
   - Redirect to /dashboard after login

2. GITHUB ACTIVITY FETCHER (githubService.js)
   - List user's repos via GET /user/repos
   - Fetch commits from last 24h: GET /repos/{owner}/{repo}/commits?since=yesterday
   - Fetch PRs: GET /repos/{owner}/{repo}/pulls?state=all&sort=updated
   - Fetch issues: GET /repos/{owner}/{repo}/issues?state=open
   - Return structured JSON: { commits: [], prs: [], issues: [] }

3. AI STANDUP GENERATOR (claudeService.js)
   Use this exact Claude prompt structure:

   System: "You are a senior engineering standup assistant. 
   Generate concise, professional daily standup updates from 
   GitHub activity data. Always respond in valid JSON only."

   User: `
   Based on this GitHub activity from the last 24 hours, generate a standup:
   
   Commits: {commits list with messages}
   Pull Requests: {pr titles and statuses}  
   Open Issues: {issue titles}
   User's tone preference: {professional/casual/bullet-points}
   
   Respond ONLY with this JSON structure, no markdown:
   {
     "yesterday": "What I did yesterday (2-3 sentences)",
     "today": "What I plan to do today (2-3 sentences)",
     "blockers": ["blocker 1 if any", "blocker 2 if any"],
     "next_priority_task": "Single most important task based on open issues/PRs",
     "velocity_score": 7  // 1-10 score of yesterday's productivity
   }
   `

4. DASHBOARD UI
   - Top: "Good morning {username}" + date
   - Repo selector dropdown
   - Big "Generate Today's Standup" button
   - StandupCard showing: Yesterday / Today / Blockers / Priority Task
   - Velocity score badge (color coded: green=8-10, yellow=5-7, red=1-4)
   - VelocityChart: bar chart of commits per day (last 7 days) using recharts
   - "Copy to Clipboard" button
   - "Send to Slack" button (if webhook configured)
   - "Regenerate" button

5. HISTORY PAGE
   - List of all past standups (date, repo, velocity score)
   - Click to expand and see full standup

6. SETTINGS PAGE
   - Slack webhook URL input + test button
   - Standup tone selector (Professional / Casual / Bullet Points)
   - Default repo selector
